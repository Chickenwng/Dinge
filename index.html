Funny Theory: How to Bypass GoGuardian with Chrome

I was messing around and found something interesting. Here's my theory:

The Discovery

When you open a new window with a URL like https://classroom.google.com using window.open() and immediately write to its DOM like this:

newWindow.document.write(html)

It causes the page to turn into an about:blank page, which some extensions (like GoGuardian) fail to block. Even though about:blank is blacklisted, the extension doesn't intervene.

The Theory

My theory is that when you write to the window while it’s still loading, Chrome doesn't inform extensions that the page has loaded. As a result, GoGuardian doesn’t block the about:blank tab.

This gives the illusion that you’ve bypassed the extension without triggering any security measures. I’m planning to write an extension to test this theory out later.

Example Code

Here's the basic code for the experiment:


window.open('https://classroom.google.com', 'newWindow');
var newWindow = window.open();
newWindow.document.write("
Hello from about:blank!
");
